# Tutorials

Tutorials are guided walkthroughs that take you from zero to a concrete outcome. Each tutorial assumes you are new to that topic and walks you through every step. Unlike how-to guides, tutorials explain what you are doing as you do it.

---

## Available tutorials

| Tutorial | Outcome | Best for |
|---|---|---|
| [Create your first repository in 5 minutes](quickstart.md) | A new GitHub repository created via the web UI | First-time repository creators |
| [Create a repository using the CLI](create-first-repository.md) | A new repository created from the command line | Developers who prefer the terminal |
| [Build your first template repository](create-first-template.md) | A template scaffolded, validated, and ready to use | Platform engineers building templates |
| [Set up RepoRoller for your organization](set-up-reporoller.md) | A fully running RepoRoller deployment | Operators doing first-time installation |
| [Tour of the web interface](tour-of-the-web-ui.md) | Familiarity with every screen in the UI | New users exploring the web interface |

---

## Recommended learning path by persona

### Repository creator

1. Start with the [web UI quickstart](quickstart.md) to create your first repository.
2. If you prefer the command line, follow [Create a repository using the CLI](create-first-repository.md).
3. Take the [Tour of the web interface](tour-of-the-web-ui.md) to understand every screen.

### Platform engineer

1. Complete the [web UI quickstart](quickstart.md) to understand the experience from a creator's perspective.
2. Follow [Build your first template repository](create-first-template.md) to create your first template.
3. Read [How configuration is resolved](../explanation/configuration-hierarchy.md) to understand the configuration system.

### Operator

1. Read [How RepoRoller works](../explanation/system-overview.md) for an architectural overview.
2. Follow [Set up RepoRoller for your organization](set-up-reporoller.md) for step-by-step deployment.
3. Review the [Environment variables reference](../reference/environment-variables.md) for production configuration.
